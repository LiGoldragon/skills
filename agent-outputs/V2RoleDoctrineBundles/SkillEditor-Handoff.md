## Observations And Tradeoffs

- Directly bundling full `reporting`, Rust, Nix, intent, privacy, secrets, and
  workspace vocabulary skills would make packets large and triggered duplicate
  heading failures. The implementation uses slim self-contained composites for
  role packet doctrine instead of dependency edges that expand into the full
  corpus.
- `rust-auditor` still includes full `architectural-truth-tests`; that module is
  large but audit-specific enough to justify the weight for now.
- `intent-translator` still includes full `bead-weaver`; it is modest in size
  and directly part of the role's normal work.
- The new composites are not active first-class skill outputs; they exist as
  role-packet source modules only.
- Primary generated outputs remain stale until a repo operator intentionally
  runs generation against the target workspace.
