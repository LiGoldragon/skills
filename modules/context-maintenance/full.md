# Skill — context maintenance

## Rules

Context maintenance reduces accumulated artifacts into their owning durable homes. Preserve useful substance; delete transcript residue, duplicate summaries, stale plans, and obsolete evidence.

Use this authority order:

- psyche intent goes through the accepted intent path;
- repository operating guidance goes to `AGENTS.md`;
- architecture facts go to `ARCHITECTURE.md`;
- project overview material goes to `README.md`;
- active work goes to the tracker;
- reusable agent discipline goes to skills only when it is not repo-specific;
- everything else is superseded or deleted.

Each artifact has one fate: keep as an active pickup surface, migrate into an owning source, or retire. Do not preserve a file merely because it exists. Do not delete before moving load-bearing facts.

Start with the lane, directory, or topic named by the task. Avoid broad workspace sweeps unless explicitly requested.

Before deleting or superseding, verify no active manifest, tracker item, prompt, or handoff still names the artifact as its pickup surface. If a reference remains, update it or leave a short supersession pointer.

A helper may inventory a bounded pile, but the lead owns disposition and durable edits. Treat helper output as evidence, not authority.
