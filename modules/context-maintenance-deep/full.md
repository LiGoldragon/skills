# Skill — deep context maintenance

## Rules

Use this only for sweeps broader than one lane: cross-lane context cleanup, lane retirement, or successor sweeps that replace an older maintenance ledger.

Organize by topic before lane. Route each topic's live substance to the newest canonical source, active tracker item, accepted intent path, or explicit abandonment. Keep only what a fresh agent still needs to act correctly.

A cross-lane sweep produces handoffs, not cross-lane edits. The dispatcher applies only changes it owns; other lanes receive recommendations with evidence and exact references.

A successor sweep retires the older sweep when it covers the same scope, re-ranks the current surface, and carries forward any still-live handoffs.

## Retiring a lane

Retire a lane only after every leftover item is routed:

1. Migrate durable rules, architecture, and project guidance into the owning files.
2. Convert implementable work into tracker items with dependencies.
3. Settle, abandon, or park pending design decisions in the relevant owning surface.
4. Delete drained working artifacts after references are removed or updated.
5. Update the retired-lane registry if the workspace uses one.

Do not create a new artifact pile to explain the cleanup. The cleaned sources and tracker state are the result.
