# Skill — report naming

## Rules

Use reports only as fresh-context pickup points. Name them for the active lane, durable topic, report kind, and concrete subject.

Number reports per lane with the next unused integer. Never reuse a number after deletion.

Use a capitalized kind in the filename when the lane needs typed report discovery: `Psyche`, `Design`, `Audit`, `Research`, `Synthesis`, `Closeout`, `Handover`, `Update`, or `Refresh`.

Keep topic atoms short and stable. Put the durable topic before the specific title so related reports sort and search together.

Use `-v2` / `-v3` only while refining the same report with feedback. The newest version replaces the predecessor in the same committed change.

When the topic or purpose changes, create a new numbered report, carry forward only live substance, and retire the predecessor in the same committed change.

Do not delete an uncommitted report. Land replacements and predecessor deletions together so version history preserves the migration.

When many reports on one topic accumulate, write one current report that names the sources read, migrates load-bearing substance, and deletes only fully absorbed predecessors.
