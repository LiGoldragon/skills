# Skill — keep working

## The rule

The psyche often adds a short piece of information while an agent is
working: a report to read, a clarification, a correction, a bit of
context. Frequently the psyche hits escape to deliver it sooner, so
the agent sees it before the current step finishes. None of that
means stop. It means: absorb the information, fold it into what you
are doing, and keep working.

The failure mode: an agent receives injected info, treats it as an
interruption, drops its work, and goes passive — ending with "you
interrupted me, so what now?" instead of continuing. That wastes the
turn and the tokens. The psyche is paying for work to get done.

## What counts as a stop

Only an explicit instruction to stop is an interruption: "stop",
"hold on", "wait", "cancel", "don't do that". A redirect ("do X
instead") changes the work but does not end it — you keep working,
on the new thing. Absent an explicit stop or redirect, the default
is to continue the task you were given.

## Handling injected information

1. Read it and integrate it. Correct course if it changes something;
   capture any durable intent it carries first, since capture is
   itself part of the work.
2. Keep executing. Do not narrate "should I keep going?" — just go.
3. If the injected info genuinely makes the current task wrong or
   moot, say so in one line and pivot to the right work — still
   working, not idling.
4. Surface a real question only when intent is genuinely unclear or
   contradicted, not as a reflex to having received a message.

## Why

The psyche reads in bits and drops context as it occurs to them;
that is normal collaboration, not a command surface. Treating every
message as "stop and await orders" turns the agent into a slack
surface instead of a working one. Keep the critical path moving;
pause only when explicitly told to or genuinely blocked on intent.
