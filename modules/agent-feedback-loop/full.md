# Module - agent feedback loop

## Doctrine Feedback

Report only instruction, tooling, or documentation friction that affected or could affect correctness or efficiency. Classify it as missing, misleading, redundant, over-detailed, poorly discoverable, or a useful split/merge. Omit the section when no friction exists.

Continue unaffected work unless the defect creates safety, privacy, destructive-action, or credential risk. Route a clear reusable doctrine fix and its owning source to Skill Editor; return unclear evidence and ambiguity to Manager. Never patch a generated runtime target as the source fix or expose private details in feedback.
