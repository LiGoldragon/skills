# Skill — helper context transfer

## Use the helper as the context carrier

When the psyche asks for a helper to preserve context, gather context, or
restart a task cleanly, the helper receives the full reading envelope. The
lead keeps only the context needed to run the Spirit gate, name the lane/report
mechanics, and write the dispatch. The helper reads the workspace rules,
skills, reports, command outputs, and source locators that the lead would
otherwise have loaded.

This is the shape for fresh-session cleanup, context handovers, report
triage, broad audits, and any request where the point of delegation is to keep
the lead's smart zone available for judgment. A thin helper brief is correct
only when the helper already has all the needed discipline or the requested
task is genuinely local.

## Build the reading envelope

The lead identifies the envelope from visible instruction surfaces rather than
reading the files:

- required-reading paths named by the psyche;
- AGENTS/INTENT/ESSENCE paths required by the workspace contract;
- skills named by the task, the role, or `skills/skills.nota`;
- source locators the helper must inspect, such as reports, beads, Spirit
  search terms, branches, commands, or repo files;
- exact psyche instructions that constrain the helper's behavior.

If a path is required reading for the lead's judgment, it is required reading
for the context-preserving helper. Put the path in the brief and tell the
helper to read it before acting.

## What the lead does before dispatch

Run the small envelope locally:

1. Classify the psyche prompt through the Spirit gate.
2. Register or identify the session lane only when needed for the dispatch.
3. Name the helper's task, boundaries, read-only/write authority, and return
   shape.
4. Include the full reading envelope and source locators in the helper brief.
5. Stop reading the delegated material.

While the helper runs, the lead waits if the psyche asked it to wait, or works
only on unrelated tasks. When the helper returns, synthesize from its distilled
result; inspect specific files only to verify or apply decisions that the
helper surfaced.

## Brief shape

A context-transfer helper brief carries:

- purpose: the task the helper is enabling;
- required reading: every file, command, and search term the lead would have
  opened;
- authority: read-only, report-writing, or specific file-edit scope;
- boundaries: what the helper must avoid deciding or editing;
- return shape: clusters, findings, open questions, file paths, commands run,
  and recommended next actions.

For a read-only explorer, ask for evidence-backed clusters and the highest
leverage psyche question. For a worker, assign disjoint write ownership and
require a changed-file list.
