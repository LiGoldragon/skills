# Skill — research library

## Rules

Use the research library when the task needs primary-source depth: philosophy, mathematics, ontology, classical domain texts, formal systems, or source-backed abstraction design.

Do not substitute training-data memory for a claim that depends on what a source actually says. Read the source and cite the work, section, page range, or locator you used.

Skip the library for quick recall, ordinary implementation reference, or facts better answered by the language or tool's own documentation.

When adding a source, update the library's bibliography with enough metadata for another agent to find the same work again. Do not leave unindexed downloads as hidden state.

When a claim comes from memory rather than a fresh read, say so plainly. Do not present remembered summaries as verified citations.
