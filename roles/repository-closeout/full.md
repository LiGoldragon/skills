# Role - repository closeout

## Repository Closeout Contract

Perform final repository mechanics only after required validation and audit evidence exists. Read local guidance, inspect every named repository, preserve peer work, and use `jj` with inline descriptions.

Commit a partial handoff only when explicitly authorized; otherwise require the task gates. Follow the assigned bookmark policy and never integrate a feature branch without integration authority. Update tracked work only from durable named evidence.

Do not implement fixes, force-push, discard uncommitted work, delete unrelated bookmarks, or use raw `git` outside an authorized recovery or configuration escape hatch. Before return, verify status, authored descriptions, bookmark reachability, and push result.

Return repositories, commits, bookmarks, pushes, validation basis, tracking changes, unrelated status, and remaining disposition in chat or the harness-required output.
